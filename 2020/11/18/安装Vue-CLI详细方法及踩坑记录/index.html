    安装Vue-CLI详细方法及踩坑记录

    发表于 2020-11-18 更新于 2021-02-22 分类于 vue
    666

    全局安装

    配置: 如果环境中没有配置全局目录,那么安装模块时会自动安装装到C:\User\当前用户\Appdata\Roaming\npm目录下

    全局目录配置

    配置时如果出现

    npm : 无法将“npm”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。请检查名称的拼写，如果包括路径，请确保路径正确，然后再试一次。
    原因: 没有权限，以管理员身份运行即可，也有可能是 nodejs 安装后没有配置环境变量

    安装时如果超级慢或者经常被卡住

    或者先

    安装时如果出现（一）

    就先用npm clean cache –force清除缓存
    再次安装

    安装时如果出现（二）


    这个玩意我也是第一次见到，清缓存等方法还会给我报错，就记录一下吧，搞了好长时间，最后参考这个解决

    1. 使用 npm config get prefix 查看默认全局安装路径
      复制默认全局安装路径在你的系统显示目录中删除npm文件夹
    2. 在终端cd到上面查询到的路径里,使用 npm init -f 初始化npm就会生成一个package.json文件
    3. 使用 npm install formidable –save 接着安装依赖
    4. 使用 npm install -g @vue/cli 安装即可


    然后配置环境变量,重启一下PowerShell,使用 vue -V 检测vue-cli版本

    安装成功

    配置环境变量

    右键我的电脑 - - - 属性 - - - 高级系统设置 - - - 环境变量
    用户变量和系统变量应该都是可以的
    区别就是用户变量是针对当前用户, 系统变量针对所有用户

    使用vue命令时出现

    把目录下的vue.ps1文件删了就行
